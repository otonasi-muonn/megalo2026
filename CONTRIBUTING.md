# コントリビューションガイドライン

このプロジェクトにご関心をお寄せいただき、ありがとうございます！バグ報告、機能要望、コードの修正、ドキュメントの改善など、あらゆる貢献を歓迎します。

スムーズにコラボレーションを進めるため、以下のガイドラインをご一読ください。

## 🐛 バグ報告と機能要望

バグを見つけた場合や、新しい機能を提案したい場合は、Issueを作成してください。

1. 新しいIssueを作成する前に、すでに似たようなIssueが存在しないか検索して確認してください。
2. 適切な **Issue テンプレート** を選択し、必要事項を記入して送信してください。

## 💻 開発環境のセットアップ

プロジェクトをローカルで動かすための基本的な手順です。（※詳細は各プロジェクトの `README.md` も参照してください）

1. このリポジトリをフォーク（Fork）します。
2. フォークしたリポジトリをローカルにクローンします。
   ```bash
   git clone https://github.com/あなたのユーザー名/リポジトリ名.git
   ```
3. 必要な依存関係をインストールします（言語やフレームワークに合わせて実行してください）。

## 🛠 開発ワークフローと Pull Request (PR)

コードの修正や機能追加を行う場合は、以下の手順で Pull Request を作成してください。

1. **ブランチの作成**: 作業を始める前に、メインのブランチから新しいブランチを作成してください。ブランチ名は作業内容が直感的にわかるように命名してください（例: `feature/add-login`, `bugfix/fix-header`）。
2. **コードの変更**: 実装や修正を行います。プロジェクトに `.editorconfig` がある場合は、そのフォーマット設定に従ってください。
3. **コミット**: 変更をコミットします。コミットメッセージは、何を変更したのか明確でわかりやすいものにしてください。
4. **テスト**: 変更によって既存の動作が壊れていないか確認してください。
5. **Push**: ローカルのブランチを、フォークした自分のリポジトリにプッシュします。
6. **PRの作成**: 本リポジトリに対して Pull Request を作成します。**PRテンプレート**が用意されているので、それに従って変更内容や動作確認の方法を記述してください。
