# 🚀 プロジェクト概要 (_complete)

## 1. コンセプト
* **一言でいうと**: スワイプで風を起こしてキャラを運ぶ物理アクション ＆ 直感操作でステージを制作・共有できるブラウザゲーム
* **解決したい課題**: 「自分だけのピタゴラ装置的なパズルを作ってみたい」という欲求の実現と、プログラミング不要で自作ステージを配信・共有できる環境の提供
* **ターゲット**: 物理演算やギミックを組み合わせるのが好きな人、視聴者参加型のゲーム実況者とリスナー

## 2. 技術スタック
| 領域 | 採用技術 | 備考 |
| --- | --- | --- |
| フロントエンド | UIはReact、ゲーム部分はCanvasでKAPLAYを動かす。useRefを用いてReactの再レンダリングからKAPLAYのライフサイクルを完全に分離する。 |
| バックエンド | Hono | 超軽量API |
| データベース | ORMは不使用。公式の @supabase/supabase-js を利用し、CLIで生成したTypeScriptの型（Database型）を適用して最速・最軽量の通信を行う。 |
| インフラ | Vercel | フロント・バックエンドのホスティング |
| 開発基盤 | pnpm workspaces | フロントエンドとバックエンドのコードを1つのリポジトリで管理し、型の完全共有を実現するモノレポ構成。 |

## 3. 機能一覧 (MVP)
初期リリース（MVP）で絶対に実装する機能をリストアップします。
- [ ] Googleログイン機能 (Supabase Auth)
- [ ] ステージ作成機能: ギミック配置、テストプレイ
- [ ] ステージ公開のクリア制限機能: テストプレイでクリアできた場合のみ、DBへの「保存（公開）」を許可するフラグ制御
- [ ] ステージプレイ機能: スワイプによる風の発生、物理演算
- [ ] ステージ保存・管理機能: DB保存、ダッシュボードでの一覧表示・編集・削除機能
- [ ] ステージ取得機能: JSONデータをDBから取得し、KAPLAY上で再構築
- [ ] シェア（URLコピー）機能: 特定のステージ画面のURL（例: /play/[id]）をクリップボードにコピーする

## 4. UI/UX・画面構成（※UIがある場合）
* **ホーム (`/`)**
  - ゲームの説明
  - ステージ一覧
  - Googleログインボタン(ダッシュボードへ)
* **ダッシュボード (`/dashboard`)**
  - 自分のステージ一覧・編集・削除
  - 新規ステージ作成ボタン（ステージ制作へ）
* **ステージ制作 (/create または /edit/[id])**
  - ギミック一覧（トゲ、ばね、送風機、波など）からのドラッグ＆ドロップ配置
  - テストプレイ（プレイ画面と同等の挙動確認）
  - 保存ボタン（ダッシュボードへ戻る）
* **プレイ画面 (/play/[id])**
  - スワイプで風を起こす（メインアクション）
  - リトライ / ポーズ
  - クリア・失敗判定（結果画面へ）
* **結果画面 (/result)**
  - クリア or 失敗の表示
  - リトライ（プレイ画面へ戻る）
  - ステージに戻る / 別のステージを選ぶ（ダッシュボードへ戻る）
