# 🚀 プロジェクト概要

## 1. コンセプト

* **コア体験**: スワイプで風を起こしてキャラを運ぶ物理アクション ＆ 直感操作で作れるステージ制作・共有ブラウザゲーム
* **目的・課題解決**: プログラミング不要で「自分だけのピタゴラ装置」を作り、手軽に配信・共有できる環境の提供
* **ターゲット**: 物理演算パズルが好きなユーザー、視聴者参加型のゲーム実況者とそのリスナー

## 2. システムアーキテクチャ・技術スタック

モノレポ構成（`pnpm workspaces`）を採用し、フロントエンド・バックエンド間の完全な型共有を前提とします。
| 領域 | 採用技術 | 用途・設計思想 |
| --- | --- | --- |
| フロントエンド | React (Vite) + KAPLAY | UIはReact、ゲーム描画はKAPLAY。`useRef`でReactの再レンダリングとゲームのライフサイクルを完全に分離。 |
| バックエンド | Hono | 超軽量APIサーバーによるエンドポイント提供。 |
| データベース | Supabase | ORM不使用。`@supabase/supabase-js`とCLI生成の型定義（Database型）を利用し、オーバーヘッドを最小化した通信。 |
| インフラ | Vercel | フロント・バックエンドのホスティング。 |

## 3. 機能一覧 (MVP)

初期リリースで提供する、ユーザーが利用できるコア機能です。

* **認証機能**: Googleアカウントによるログイン (Supabase Auth)
* **ステージプレイ機能**: スワイプによる風の発生・物理演算を用いたギミック攻略
* **ステージ制作機能**: ドラッグ＆ドロップによるギミック（トゲ、ばね、送風機、波など）の配置とテストプレイ
* **クリア制限付き公開機能**: 制作者自身がテストプレイでクリアできた場合のみ、公開（DB保存）を許可する制御フラグ
* **マイページ機能**: 自作ステージの一覧表示、編集、削除
* **シェア機能**: 特定ステージへのダイレクトリンク（`/play/[id]`）のクリップボードコピー

## 4. 画面構成・ルーティング

各画面の役割と、提供するUI要素を定義します。

* **`/` (ホーム)**: ゲーム説明、公開済みステージ一覧、ログインへの導線
* **`/dashboard` (ダッシュボード)**: 自分のステージ一覧・管理、新規作成への導線
* **`/create`, `/edit/[id]` (ステージ制作)**: ギミック配置UI、テストプレイ実行、保存（公開）実行
* **`/play/[id]` (プレイ画面)**: メインアクション（風を起こす）、リトライ・ポーズ操作
* **`/result` (結果画面)**: クリア/失敗判定の表示、リトライ、別ステージへの遷移
